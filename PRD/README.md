# PRD – Plateforme interne 4Runners

## Vision
Créer un outil interne ultra efficace pour une agence digitale,
destiné à centraliser les échanges client, la validation, la production
et la documentation, sans devenir un ClickUp / Notion / Slack bis.

L’objectif est de réduire le bruit, les outils dispersés
et les pertes de temps liées à la coordination.

## Utilisateurs
- Agence (utilisateur principal)
- Client (utilisateur secondaire, accès limité et guidé)

## Problème à résoudre
- Trop d’échanges dispersés (email, WhatsApp, ClickUp, Drive)
- Manque de visibilité sur l’état réel des actions
- Validation client lente ou floue
- Difficulté à centraliser la connaissance (briefs, process, décisions)

## Piliers fonctionnels
- Inbox centrale event-driven (tout converge ici)
- Espaces clients modulaires (features ON/OFF)
- Validation plutôt que gestion de tâches
- Documentation structurée et simple
- UX rapide, peu d’écrans, peu de clics

## Ce que la plateforme N’EST PAS
- Un gestionnaire de tâches complexe
- Un ClickUp bis
- Un Notion bis
- Un CRM
- Un outil collaboratif temps réel

## Règles produit non négociables
- L’agence est toujours prioritaire sur l’UX
- Le client voit uniquement ce qui le concerne
- Module OFF = invisible + inaccessible
- Pas de complexité sans ROI clair
- Chaque fonctionnalité doit améliorer le quotidien de l’agence

## Succès du produit
- L’agence peut piloter sa journée sans sortir de l’outil
- Le client comprend quoi faire sans formation
- Les validations sont traçables
- La connaissance n’est plus perdue
